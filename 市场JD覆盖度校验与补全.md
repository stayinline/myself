# 市场 JD 覆盖度校验与补全（解决"简历驱动漏题"问题）

> 生成日期：2026-06-05 · 基于猎聘/拉勾等 2026 年真实 JD 采样
> 解决的问题：之前的 100 题与学习计划是**简历驱动**的，只覆盖"你写过的"，会漏掉"市场要、但你简历没写"的盲区（unknown unknowns）。本文给出**永久解决方法 + 本次补全清单**。

---

## 一、为什么会漏？（先讲清根因）

| 出发点 | 能覆盖 | 会漏掉 |
|---|---|---|
| **简历驱动**（之前） | 你写过的 = 已知的已知 | 市场要、你没写的 = **盲区** |
| **JD 驱动 / 市场驱动**（应改用） | 目标岗位真实要求全集 | 几乎不漏（再用复测补） |

**结论**：自测题应该从**目标 JD**反推，而不是从简历正推。简历只决定"你现在在哪"，JD 决定"你要去哪"。

---

## 二、永久解决方法：JD 驱动的"反向校验"四步法（可反复用）

> 这是一套你以后每季度都能自己跑一遍的机制，确保计划永远对齐市场。

### Step 1：精确定义目标（先锁定靶子）
- 目标岗位（如：实时数仓/大数据开发、数据+AI、AI Infra、RAG 数据工程师）
- 目标层级（资深/专家/架构）、目标城市、目标公司梯队（大厂/独角兽/中厂）
- 目标薪资区间（反推要求强度）

### Step 2：采集真实 JD（20–30 条）
- 渠道：猎聘、拉勾、BOSS、目标公司官网、脉脉。
- 每条只抄"任职要求 + 岗位职责"两段，存进一个表格。
- 覆盖不同公司/梯队，避免单一来源偏差。

### Step 3：关键词词频统计（让数据说话）
- 把所有 JD 的技能关键词抽出来，统计**出现频率**。
- 频率 = 市场权重：出现 ≥70% 的是"硬门槛"，30–70% 是"高频加分"，<30% 是"长尾/可选"。
- 这一步直接产出"市场要什么、要多强"的客观排序。

### Step 4：差集 + 分诊 + 打补丁
- 把词频表与你现有计划/能力做差集，分三类：✅已覆盖 / ⚠️浅覆盖 / ❌缺失。
- 对 ❌/⚠️ 做**分诊**：必补（高频硬门槛）/ 认知级（了解即可）/ 可跳过（不符合你定位）。
- 把"必补"插进学习计划，"认知级"安排轻量学习，"可跳过"明确放弃理由。

### 持续机制（关键）
- **每季度复跑一次**（JD 会漂移，尤其 AI 侧半年一变）。
- **每次面试后回填**：被问到却没准备的点，立刻补进词频表——这是最真实的市场反馈。
- **二次自测对照**：完成计划后重测，仍弱的 + 新出现的 = 下一轮输入。

## 四、缺口分诊（结合你的定位：数据为主、AI 差异化、不与算法卷）

> 不是所有缺口都要补。按你的护城河定位分三档：

### 🔴 必补（高频硬门槛，且符合你定位）
1. **Python**：AI/数据侧的"普通话"，RAG/AI 岗硬门槛。你 Java 强，但 Python 必须能熟练写数据处理与 AI 链路。
2. **文档解析 / 非结构化数据处理**（PDF/表格/OCR 清洗）：这是 RAG **数据链路**的核心，**正好是你数据工程师的主场**，最该吃下的差异化。
3. **Elasticsearch 深入**（倒排/分词/打分/混合检索）：RAG 岗硬门槛，你有 ES 基础，补深性价比高。
4. **Hadoop 生态认知**（HDFS/YARN/HBase）+ **Presto/Trino**：大数据岗高频门槛，认知级即可消除过滤。
5. **调度（Airflow/DolphinScheduler）+ 数据同步（DataX/SeaTunnel）**：ETL 链路标配，你已有 Airflow 底子。

### 🟡 认知级（能讲清概念/取舍即可，不深陷）
6. **特征平台 Feature Store** + **LLMOps（MLflow/模型注册/评测流水线）**：AI Infra 方向的体系认知。
7. **推理加速（vLLM/Triton/量化/KV Cache）**：知道是什么、解决什么、何时用，**不写算子**。
8. **LLM 基础（Transformer/SFT/LoRA）**：理解原理与微调概念，能对话，**不亲自训练**。
9. **多向量库（pgvector/Qdrant）+ LangChain/LlamaIndex**：知道差异与选型。
10. **Delta Lake、数据安全/权限/脱敏**：补齐数据湖与架构岗的认知短板。

### ⚪ 可跳过（不符合定位，明确放弃）
- CUDA/算子优化、GPU 硬件深度、分布式训练（DeepSpeed）、顶会论文——这是**算法/AI Infra 专家**的活，与你"数据+AI"定位投入产出比极低（职业规划文档已论证）。
- **知识图谱/GraphRAG** 介于 🟡 和 ⚪：若你目标含 RAG 进阶岗则升为 🟡 认知级，否则可暂放。

## 五、计划补丁建议：扩展为"9 周核心 + 第 10–11 周市场补全"

不打乱原 9 周（基本盘+旗舰），在其后追加两周专补市场盲区：

### 第 10 周｜数据工程广度补全（消除过滤点）
- D64 Python for Data/AI：语法速通 + 用 Python 写一个数据清洗 + 调用 Embedding（衔接 AI）
- D65 Hadoop 生态认知：HDFS/YARN/MapReduce 原理 + HBase 适用场景（能讲清即可）
- D66 查询引擎：Presto/Trino 架构与"为什么需要联邦查询"，对比 ClickHouse
- D67 调度与数据同步：Airflow DAG 实操 + DataX/SeaTunnel 采集认知
- D68 数据安全与治理补全：权限/脱敏/加密、数据资产、Delta Lake 认知
- D69 复盘：输出《大数据生态广度笔记》

### 第 11 周｜AI 工程化盲区补全（差异化加固）
- D70 文档解析流水线：PDF/Word/表格/OCR 清洗 → 分块 → 入向量库（你的主场，做深）
- D71 Elasticsearch 深入：倒排/分词/打分 + ES 向量 + 关键词×向量混合检索 + RRF
- D72 多向量库选型：pgvector / Qdrant / Weaviate vs Milvus；LangChain/LlamaIndex 框架与"何时脱离框架"
- D73 LLM 基础认知：Transformer/Embedding/SFT/LoRA 原理（能对话，不训练）
- D74 AI Infra 体系：Feature Store + LLMOps（MLflow/模型注册/评测/监控）+ 推理加速（vLLM/量化）认知
- D75 复盘 + 二次自测（含本次新增盲区）：更新方法论与简历

> 这样总计 **11 周 / 75 天**，既保住核心深度，又补齐市场广度。
