## 一、实验：同一任务对比三种 Prompt 技术

### 2.2 Pydantic 校验模型

```python
from pydantic import BaseModel, Field, field_validator
from typing import Optional, Literal
from datetime import datetime

class CustomerFeedback(BaseModel):
    """客户反馈结构化模型 - 用于校验 LLM 输出"""

    customer_name: Optional[str] = Field(None, description="客户姓名")
    phone: Optional[str] = Field(None, description="联系电话")
    order_id: Optional[str] = Field(None, pattern=r"^ORD-\d{4}-\d+$")
    product: Optional[str] = Field(None, description="产品名称")
    amount: Optional[float] = Field(None, ge=0, description="订单金额")
    purchase_date_relative: Optional[str] = None
    issue_type: Optional[Literal[
        "quality_defect", "logistics", "service",
        "complaint", "praise", "other"
    ]] = None
    issue_description: Optional[str] = None
    service_rep: Optional[str] = None
    resolution_offered: Optional[str] = None
    sentiment: Literal["positive", "negative", "mixed", "neutral"]
    urgency: Literal["high", "medium", "low"]

    @field_validator("phone")
    @classmethod
    def validate_phone(cls, v):
        if v is not None and len(v) < 7:
            raise ValueError(f"电话号码过短: {v}")
        return v

    @field_validator("amount")
    @classmethod
    def validate_amount(cls, v):
        if v is not None and v > 1000000:
            raise ValueError(f"金额异常大，可能是单位错误: {v}")
        return v
```

### 2.3 带重试的结构化提取函数

```python
import json
import logging
from typing import Tuple, Optional

logger = logging.getLogger(__name__)

def extract_structured_feedback(
    text: str,
    llm_client,
    max_retries: int = 3
) -> Tuple[Optional[CustomerFeedback], dict]:
    """
    从文本中提取结构化客户反馈，含校验和重试。
    返回: (解析结果, 元数据)
    """
    metadata = {
        "attempts": 0,
        "parse_errors": [],
        "validation_errors": [],
        "total_tokens": 0,
        "success": False,
    }

    error_context = ""

    for attempt in range(max_retries):
        metadata["attempts"] += 1

        # 构建 prompt（追加之前的错误信息帮助修正）
        prompt = PROMPT_COT_SCHEMA.format(text=text)
        if error_context:
            prompt += f"\n\n## 上次输出的错误\n{error_context}\n请修正以上问题后重新输出。"

        # 调用 LLM
        try:
            response = llm_client.chat(
                messages=[{"role": "user", "content": prompt}],
                response_format={"type": "json_object"},
                temperature=0.1  # 低温度提高确定性
            )
            raw_output = response.content
            metadata["total_tokens"] += response.usage.total_tokens
        except Exception as e:
            logger.error(f"LLM call failed: {e}")
            metadata["parse_errors"].append(str(e))
            continue

        # Step 1: JSON 解析
        try:
            parsed = json.loads(raw_output)
        except json.JSONDecodeError as e:
            error_msg = f"JSON解析失败: {str(e)}"
            metadata["parse_errors"].append(error_msg)
            error_context = error_msg
            logger.warning(f"Attempt {attempt+1}: {error_msg}")
            continue

        # Step 2: Pydantic 校验
        try:
            validated = CustomerFeedback(**parsed)
            metadata["success"] = True
            logger.info(f"Extraction succeeded on attempt {attempt+1}")
            return validated, metadata
        except Exception as e:
            error_msg = f"字段校验失败: {str(e)}"
            metadata["validation_errors"].append(error_msg)
            error_context = error_msg
            logger.warning(f"Attempt {attempt+1}: {error_msg}")
            continue

    # 所有重试都失败
    logger.error(f"All {max_retries} attempts failed for text: {text[:50]}...")
    return None, metadata
```

### 3.2 Flink + LLM 集成模式

```python
# 伪代码：Flink UDF 中调用 LLM 提取
class LLMExtractUDF(TableFunction):
    """Flink Table API UDF：批量调用 LLM 做结构化提取"""

    def __init__(self, api_endpoint, rate_limit=10):
        self.api_endpoint = api_endpoint
        self.rate_limiter = RateLimiter(rate_limit)  # 控制并发
        self.cache = LRUCache(maxsize=10000)  # 缓存相同输入

    def eval(self, raw_text: str):
        # 1. 检查缓存
        cache_key = hashlib.md5(raw_text.encode()).hexdigest()
        if cache_key in self.cache:
            return self.cache[cache_key]

        # 2. 限流等待
        self.rate_limiter.acquire()

        # 3. 调用提取函数
        result, metadata = extract_structured_feedback(
            raw_text, self.llm_client
        )

        # 4. 处理结果
        if result:
            output = result.model_dump_json()
        else:
            output = json.dumps({"_extraction_failed": True})

        # 5. 写入缓存
        self.cache[cache_key] = output
        return output
```

### 3.3 批处理 vs 流处理选择

| 维度 | 批处理 | 流处理 (Flink) |
|------|--------|---------------|
| 适用场景 | 历史数据回填 | 实时反馈处理 |
| 吞吐量 | 高（可并行） | 受限于 API Rate Limit |
| 延迟 | 分钟~小时级 | 秒~分钟级 |
| 成本效率 | 高（可批量折扣） | 低（持续运行） |
| 容错 | 简单（重跑即可） | 复杂（需要 checkpoint） |
| 推荐 | 存量数据处理 | 增量实时处理 |

## 四、面试话术："怎么让 LLM 稳定输出可被程序解析的结果"

### 4.1 推荐回答

> "我做了三层保障来确保 LLM 输出的可解析性：
> 
> 第一层是 API 层面的格式约束，使用 OpenAI 的 JSON Mode 或 Structured Outputs 功能，让模型在解码层面就被限制为输出合法 JSON；
> 
> 第二层是 Prompt 层面，在 system prompt 中明确给出 JSON Schema 定义和 few-shot 示例，让模型理解期望的输出结构；
> 
> 第三层是运行时校验，用 Pydantic 定义数据模型做严格校验，不通过就把错误信息追加到 prompt 里让模型修正，最多重试 3 次。
> 
> 实测下来，裸调用的可用率只有 68%，加上三层保障后达到 98%，平均只需 1.24 次调用就成功，额外 Token 成本只增加了 28%。这个方案和我们的数据管道无缝衔接——LLM 提取出的结构化数据直接写入 StarRocks/Paimon，下游 BI 和告警系统可以直接消费。"

### 4.2 进阶追问准备

**Q: 如果模型连续三次都输出不合法的 JSON 怎么办？**
> "走降级路径。具体有三种策略：一是把原始文本存入'待人工处理'队列；二是用正则表达式做 best-effort 提取，标记置信度为低；三是换一个更强的模型重试一次。选择哪种取决于业务容忍度和成本预算。在我们的系统中，大约 2% 的请求会走降级路径，日均约 200 条，由运营人员半天内处理完。"

**Q: JSON Schema 约束会不会影响模型的推理质量？**
> "会有轻微影响。Structured Outputs 的 strict mode 限制了某些采样自由度，可能导致模型在边界情况下选择'安全但不精确'的值。我的做法是：对于关键字段（如金额、状态码）用 strict 约束，对于开放性字段（如问题描述）保持灵活。同时通过评测集监控质量变化，确保约束没有显著降低内容质量。"

**Q: 大量文本需要提取时怎么处理？**
> "分两种情况：历史存量数据用批处理，开多个 worker 并行调用 LLM API，做好限流和断点续传；实时增量数据用 Flink 流处理，在 UDF 中调用 LLM，配合 Rate Limiter 控制并发。关键是做好缓存——很多反馈模板相似，缓存命中率能到 30-40%，大幅降低成本。我们处理了 50 万条历史反馈，用了 3 天时间，成本约 ¥2000。"

from pydantic import BaseModel, Field
from typing import Optional, Literal
from openai import OpenAI

class FeedbackExtraction(BaseModel):
    """结构化输出模型 - OpenAI 会严格约束输出格式"""
    customer_name: Optional[str] = Field(description="客户姓名")
    phone: Optional[str] = Field(description="联系电话")

    product: Optional[str] = Field(description="产品名称")
    amount: Optional[float] = Field(description="金额")
    issue_type: Optional[Literal[
        "quality_defect", "logistics", "service", "other"
    ]] = Field(description="问题类型")
    sentiment: Literal["positive", "negative", "mixed", "neutral"]
    urgency: Literal["high", "medium", "low"]

client = OpenAI()
completion = client.beta.chat.completions.parse(
    model="gpt-4o-2024-08-06",
    messages=[
        {"role": "system", "content": "从客户反馈中提取结构化信息。"},
        {"role": "user", "content": text}
    ],
    response_format=FeedbackExtraction,
)
result: FeedbackExtraction = completion.choices[0].message.parsed
# result 直接是 Pydantic 对象，100% 合法（API层面保证）
