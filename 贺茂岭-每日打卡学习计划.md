## 〇、重要前提：先把简历"去水"

注水简历的最大风险是：**靠夸大拿到面试，却在追问中暴露**，反而砸掉口碑。配合本计划，建议同步做两件事：

1. **短期**：把简历里你目前是 🟡/❌ 的"精通/重点"措辞**降级为"熟悉/了解/参与"**，避免被一戳就破。
2. **中期**：用这 9 周把关键项**真正补到 ✅**，再把简历"加回去"——这时是有底气的真实能力，不是水分。

> 原则：**简历可以包装，但每一条都要扛得住追问。** 本计划就是帮你把"扛不住的"变成"扛得住的"。

## 二、市场需求 → 你的差距（决定学习优先级）

2026 年"数据 / 数据+AI"岗位的高频考点权重（综合大厂 + 中厂 JD 与面试）：

| 市场考点 | 市场权重 | 你的现状 | 优先级 |
|---|---|---|---|
| Flink 深度（窗口/状态/容错/调优/SQL） | ⭐⭐⭐⭐⭐ 必考 | 有洞 | **P0** |
| 数仓建模 + 维度建模 + 数据治理 | ⭐⭐⭐⭐⭐ 必考 | 弱 | **P0** |
| Kafka | ⭐⭐⭐⭐ 高频 | 扎实 | P2 提纯 |
| OLAP（ClickHouse/Doris/StarRocks） | ⭐⭐⭐⭐ 高频 | CH 强、SR 缺 | P1 |
| 离线数仓（Spark/Hive） | ⭐⭐⭐⭐ 多数 JD 要求 | **空白** | **P0 防过滤** |
| 数据湖（Iceberg/Paimon/Hudi） | ⭐⭐⭐ 加分 | 很弱 | P1 |
| RAG / 向量库工程化 | ⭐⭐⭐ 差异化加分（增长快） | 会用不会调 | P1 差异化 |
| Agent / 多智能体 | ⭐⭐ JD 含金量参差 | 偏浅 | P2 轻量 |

**结论（学习顺序的逻辑）**：
```
P0 基本盘扎实(Flink+建模) + 防过滤补缺(Spark/Hive)
  → P1 OLAP/数据湖加分 + AI 工程化差异化
  → P2 Kafka/Agent 提纯打磨
```
> 为什么数据在前、AI 在后？因为**任何数据岗的第一道门槛都是数据基本盘**；AI 是你的"加分差异化"，但若基本盘有洞，AI 再亮也过不了初面。先把地基筑牢，AI 才能成为放大器。

## 三、学习原则

1. **费曼输出**：每个主题学完写一段"能脱稿讲给面试官"的解释，讲不清=没学会。
2. **动手 > 看文档**：Flink / Spark / Paimon / Milvus 都要本地或测试环境跑，看真实行为。
3. **对齐 JD**：每周对照 3 条真实目标 JD，确认补的点是市场要的，不做无用功。
4. **每周一产出**：周日整理成笔记，9 周攒出可对外讲的作品与方法论。
5. **时间**：工作日 1–1.5h，周末 2–3h。完不成顺延，保节奏。
